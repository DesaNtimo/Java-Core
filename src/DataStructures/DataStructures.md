# Структуры данных в Java: Исчерпывающее руководство

## 1. Иерархия типов и суперкласс Object
В Java строгая система типов. Все типы делятся на две категории:
1. **Примитивные (Primitive):** `byte`, `short`, `int`, `long`, `float`, `double`, `boolean`, `char`. Хранятся в стеке (или как часть объекта в куче), не имеют методов.
2. **Ссылочные (Reference):** Массивы, классы, интерфейсы, `String`. Указывают на объекты в куче (Heap).

Все ссылочные типы неявно наследуют класс `Object`.

### Контракт `equals()` и `hashCode()`
Корректная работа коллекций, использующих хэширование (например, `HashMap`, `HashSet`), напрямую зависит от соблюдения контракта этих методов:
* **Контракт `equals`:** Должен быть рефлексивным (`x.equals(x)`), симметричным (`x.equals(y) == y.equals(x)`), транзитивным и консистентным.
* **Контракт `hashCode`:**
  1. Если `equals` возвращает `true`, то `hashCode` обоих объектов **обязан** быть одинаковым.
  2. Если `hashCode` объектов равен, объекты **не обязательно** равны по `equals` (возникает коллизия).
* **По умолчанию:** `hashCode` является `native` методом и возвращает `identity hash code` (часто вычисляется из адреса в памяти или случайного числа). `equals` по умолчанию сравнивает ссылки (`==`).

## 2. Generics (Дженерики)
Дженерики (`<T>`) переносят проверку типов из рантайма на этап компиляции, избавляя от ручного кастования (`ClassCastException`).

* **Type Erasure (Стирание типов):** Дженерики существуют только для компилятора. При компиляции параметры типов удаляются (стираются) и заменяются на их границы (обычно `Object`). Для сохранения полиморфизма компилятор генерирует `bridge`-методы и добавляет инструкции `CHECKCAST`.
* **Массивы vs Дженерики:**
  * Массивы **ковариантны**: `String[]` — это подтип `Object[]`. Это позволяет положить `Integer` в `String[]` на этапе компиляции, но выбросит `ArrayStoreException` в рантайме.
  * Дженерики **инвариантны**: `List<String>` не является подтипом `List<Object>`. Это гарантирует 100% типобезопасность.
* **Wildcards (Маски) и правило PECS (Producer Extends, Consumer Super):**
  * `? extends T` (Ковариантность): Позволяет **только читать** из коллекции. Используется, когда коллекция является поставщиком данных (Producer).
  * `? super T` (Контрвариантность): Позволяет **только писать** в коллекцию (чтение вернет `Object`). Используется, когда коллекция является потребителем (Consumer).
* **Ограничения:** Нельзя использовать примитивы (`List<int>` запрещен), инстанцировать `new T()`, создавать массивы `new T[]` и использовать `instanceof` с дженериками (кроме `List<?>`).

## 3. Java Collections Framework (JCF)
Единая архитектура для представления и манипулирования коллекциями. Базируется на интерфейсах `Iterable` -> `Collection` и независимом интерфейсе `Map`.

### Итераторы и Fail-Fast поведение
* `Iterator<E>`: Обеспечивает безопасное удаление элементов (`remove()`) во время обхода.
* `ListIterator<E>`: Двунаправленный итератор для списков (позволяет идти назад, узнавать индекс, заменять через `set()` и добавлять `add()`).
* **Fail-Fast:** Большинство неконкурентных коллекций (например, `ArrayList`, `HashMap`) отслеживают структурные изменения через счетчик `modCount`. Если коллекция изменена не через итератор во время обхода, выбрасывается `ConcurrentModificationException`.

## 4. Списки (List)
Упорядоченные коллекции, допускающие дубликаты. Обеспечивают доступ по индексу.

* **ArrayList:** Базируется на динамическом массиве. Емкость по умолчанию — 10, при заполнении увеличивается в 1.5 раза.
  * *Сложность:* `get`/`set` — **O(1)**. `add` (в конец) — **O(1)** амортизировано. Вставка/удаление из середины — **O(N)** (требует сдвига массива).
  * *Практика:* Используется в 99% случаев благодаря локальности данных в памяти (процессорный кэш работает эффективно).
* **LinkedList:** Базируется на двусвязном списке. Реализует также интерфейс `Deque`.
  * *Сложность:* Добавление/удаление в начале и конце — **O(1)**. Доступ по индексу — **O(N)**.
  * *Практика:* Практически не используется из-за огромного оверхеда на хранение узлов и промахов процессорного кэша.

## 5. Множества (Set)
Коллекции, гарантирующие уникальность элементов. Все стандартные множества работают поверх соответствующих реализаций `Map`.

* **HashSet:** Под капотом использует `HashMap` (значениями выступает объект-заглушка). Не гарантирует порядок элементов.
  * *Сложность:* `add`, `remove`, `contains` — **O(1)**.
* **LinkedHashSet:** Сохраняет порядок добавления элементов благодаря пронизывающему двусвязному списку.
  * *Сложность:* **O(1)**, требует чуть больше памяти, чем `HashSet`.
* **TreeSet:** Базируется на `TreeMap` (Красно-черное дерево). Хранит элементы в отсортированном виде (Natural Ordering или через `Comparator`).
  * *Сложность:* Базовые операции — **O(log N)**.

## 6. Словари (Map)
Хранят пары "ключ-значение". Ключи уникальны.

* **HashMap:** Хэш-таблица. Массив "корзин" (buckets), где каждая корзина — это связный список. С Java 8, если список в корзине превышает 8 элементов, он трансформируется в Красно-черное дерево (для защиты от коллизий).
  * *Сложность:* `get`, `put`, `remove` — **O(1)** в среднем (в худшем случае **O(log N)** для дерева).
* **LinkedHashMap:** Поддерживает порядок вставки или порядок доступа (LRU-кэш).
* **TreeMap:** Красно-черное дерево. Ключи отсортированы.
  * *Сложность:* Базовые операции — **O(log N)**.

---

## 7. Очереди (Queue и Deque)
* **PriorityQueue:** Очередь с приоритетами на базе структуры "Min-Heap" (Двоичная куча). Элементы извлекаются не в порядке FIFO, а по приоритету (наименьший элемент выходит первым).
  * *Сложность:* `offer`, `poll` — **O(log N)**. `peek` — **O(1)**.
* **ArrayDeque:** Двусторонняя очередь на базе кольцевого массива. Идеальная замена устаревшему `Stack`. Позволяет добавлять/удалять с обоих концов за **O(1)**. Не допускает `null`.

---

## 8. Bitmap-структуры (Оптимизированные коллекции)
Для достижения максимальной производительности при работе с флагами и перечислениями используются побитовые операции над типом `long`.

* **EnumSet:** Высокооптимизированная реализация для `enum`. Если элементов < 64, весь набор хранится в одной переменной `long` в виде битовой маски. Всегда используйте `EnumSet` при работе с перечислениями!
  * *Сложность:* Все операции — **O(1)** (одна машинная инструкция).
* **BitSet:** Динамически расширяемый массив битов (под капотом `long[]`). Используется для флагов, побитовых операций (`AND`, `OR`, `XOR`) и плотных целочисленных множеств. Установка/чтение бита занимает **O(1)**.

## 9. Утилиты и Суррогатные коллекции
Стандартная библиотека предоставляет инструменты для работы с массивами и коллекциями:

* **Класс `java.util.Collections`:**
  * Содержит алгоритмы: `sort` (**O(N log N)** - TimSort), `binarySearch` (**O(log N)**), `reverse`, `shuffle`.
  * Фабрики: `emptyList()`, `singletonList()`.
  * Обертки (Views): `unmodifiableList()` (предотвращает прямое изменение коллекции, но отражает изменения оригинала).
* **Класс `java.util.Arrays`:**
  * Утилиты для примитивных массивов: сортировка (Dual-Pivot Quicksort), поиск, копирование.
  * `Arrays.asList()`: создает фиксированного размера `List` поверх массива (изменения отражаются на массиве, структурные изменения запрещены).
* **Неизменяемые (Immutable) фабрики (Java 9+):**
  * `List.of()`, `Set.of()`, `Map.of()`. Возвращают полностью неизменяемые коллекции. Попытка добавить `null` или изменить данные приводит к Exception. Идеальны для создания констант и возврата безопасных данных.
