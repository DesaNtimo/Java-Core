# Структуры данных в Java: Исчерпывающее руководство

## 1. Иерархия типов и суперкласс Object
В Java строгая система типов. Все типы делятся на две категории:
1. **Примитивные (Primitive):** `byte`, `short`, `int`, `long`, `float`, `double`, `boolean`, `char`. Хранятся в стеке (или внутри объектов в куче), не имеют методов.
2. **Ссылочные (Reference):** Массивы, классы, интерфейсы, `String`. Указывают на объекты в куче (Heap).

Все ссылочные типы неявно наследуют класс `Object`, получая базовый набор методов (`getClass()`, `toString()`, `clone()`, `finalize()`, методы монитора `wait`/`notify`).

### Контракт `equals()` и `hashCode()`
Корректная работа коллекций, использующих хэширование (`HashMap`, `HashSet`), критически зависит от контракта этих методов:
* **Контракт `equals`:** Должен быть рефлексивным (`x.equals(x)`), симметричным, транзитивным, консистентным и возвращать `false` при сравнении с `null`.
* **Контракт `hashCode`:**
  1. Если `equals` возвращает `true`, то `hashCode` обоих объектов **обязан** совпадать. Нарушение этого правила ведет к "потере" объектов в хэш-таблицах и утечкам памяти.
  2. Если `hashCode` равен, объекты **не обязательно** равны (коллизия). Хорошая хэш-функция минимизирует коллизии.
* **По умолчанию:** `hashCode` (native) возвращает `identity hash code` (основан на адресе в памяти). `equals` сравнивает ссылки (`==`).

## 2. Generics (Дженерики): Полное руководство
Дженерики (`<T>`) переносят проверку типов на этап компиляции, устраняя ручное кастование и `ClassCastException`.

### Механика и контракты
* **Type Erasure (Стирание типов):** Дженерики существуют только для компилятора. В байткоде они стираются до своих границ (обычно `Object`). Для сохранения полиморфизма генерируются `bridge`-методы, а при чтении компилятор вставляет `CHECKCAST`.
* **Массивы vs Дженерики:**
  * Массивы **ковариантны** (`String[]` — подтип `Object[]`). Это чревато `ArrayStoreException` в рантайме.
  * Дженерики **инвариантны** (`List<String>` не является подтипом `List<Object>`). Это гарантирует типобезопасность.
* **Правило PECS (Producer Extends, Consumer Super):**
  * `? extends T` (Ковариантность): Читать можно, писать нельзя. Используется для поставщиков данных.
  * `? super T` (Контрвариантность): Писать можно, чтение деградирует до `Object`. Используется для потребителей.

### Продвинутые концепции
* **Raw Types (Сырые типы):** Использование `List` вместо `List<T>`. Отключает проверку типов. Строго запрещено в современном коде.
* **Multiple Bounds (Множественные ограничения):** `<T extends Number & Comparable<T>>`. Класс всегда указывается первым, затем интерфейсы.
* **Heap Pollution (Загрязнение кучи):** Возникает при смешивании `varargs` и дженериков. Решается аннотацией `@SafeVarargs`, если метод не модифицирует массив аргументов.
* **Ограничения:** Запрещено использовать примитивы (`List<int>`), инстанцировать `new T()`, создавать массивы `new T[]` и использовать `instanceof` (кроме маски `List<?>`).

## 3. Java Collections Framework (JCF)
Единая архитектура на базе интерфейсов `Iterable` -> `Collection` и независимого интерфейса `Map`.

### Итераторы и механизмы обхода
* `Iterator<E>`: Обеспечивает безопасное удаление (`remove()`) во время обхода.
* `ListIterator<E>`: Двунаправленный итератор для списков (`previous()`, `nextIndex()`, `set()`, `add()`).
* `Spliterator<E>`: Разделяемый итератор (Java 8). Основа Stream API. Разделяет коллекцию (`trySplit()`) для параллельной обработки.
* **Fail-Fast поведение:** Стандартные коллекции отслеживают структурные изменения через `modCount`. Модификация в обход итератора вызывает `ConcurrentModificationException`.

## 4. Списки (List)
Обеспечивают доступ по индексу.

* **ArrayList:** Динамический массив (емкость 10, рост x1.5).
  * *Сложность:* `get`/`set` — **O(1)**. `add` в конец — **O(1)** амортизировано. Вставка/удаление в середине — **O(N)**.
  * *Особенности:* Методы `ensureCapacity(n)` и `trimToSize()` для ручного управления памятью. `subList` возвращает представление (View), а не копию данных.
* **LinkedList:** Двусвязный список. Заменяет `Deque`.
  * *Сложность:* Операции на концах — **O(1)**. Доступ по индексу — **O(N)**.
  * *Особенности:* Вставка/удаление через `ListIterator` работает за **O(1)**. Страдает от промахов процессорного кэша.

## 5. Множества (Set)
Гарантируют уникальность. Под капотом используют соответствующие `Map`.

* **HashSet:** На базе `HashMap`.
  * *Сложность:* Базовые операции — **O(1)**.
  * *Теория множеств:* Поддерживает объединение (`addAll`), пересечение (`retainAll`) и разность (`removeAll`) за **O(N)**.
* **LinkedHashSet:** Сохраняет порядок вставки благодаря двусвязному списку.
* **TreeSet:** На базе `TreeMap`. Отсортировано.
  * *Сложность:* **O(log N)**.
  * *Навигация (NavigableSet):* Поддерживает методы `lower()`, `higher()`, `floor()`, `ceiling()`, `subSet()`.

---

## 6. Словари (Map)
Пары "ключ-значение".

* **HashMap:** Хэш-таблица. Разрешает коллизии методом цепочек. При размере корзины >= 8 список трансформируется в Красно-черное дерево.
  * *Сложность:* **O(1)** в среднем.
  * *Особенности:* Поддерживает дефолтные методы Java 8: `putIfAbsent`, `getOrDefault`. Метод `containsValue` работает за **O(N)**.
* **LinkedHashMap:** Поддерживает порядок вставки ИЛИ порядок доступа (идеально для LRU-кэшей).
* **TreeMap:** Красно-черное дерево. Ключи отсортированы.
  * *Навигация:* Позволяет получать диапазоны `headMap`, `tailMap` и граничные `Entry`.

## 7. Очереди (Queue и Deque)
* **API Очередей:** Методы делятся на выбрасывающие исключения (`add`, `remove`, `element`) и возвращающие статус/null (`offer`, `poll`, `peek`). Всегда предпочтительнее вторые.
* **PriorityQueue:** Очередь с приоритетами ("Min-Heap").
  * *Сложность:* `offer`, `poll` — **O(log N)**. `peek` — **O(1)**.
  * *Внимание:* Итератор `PriorityQueue` не гарантирует обход в отсортированном порядке!
* **ArrayDeque:** Кольцевой массив. Работает с обоими концами за **O(1)**. Официальная и самая быстрая замена устаревшему классу `Stack` (LIFO). Не допускает `null`.

---

## 8. Bitmap-структуры (Оптимизированные коллекции)
Работают через побитовые операции над `long`.

* **EnumSet:** Для `enum`. До 64 элементов хранится в одной маске `long`. Скорость всех операций — 1 процессорная инструкция (**O(1)**).
* **BitSet:** Динамический битовый вектор (`long[]`). Используется для флагов и криптографии. Поддерживает логические операции `AND`, `OR`, `XOR`.

## 9. Утилиты, алгоритмы и фабрики
Инструментарий стандартной библиотеки (без учета внешних библиотек вроде Guava).

* **Класс `java.util.Collections`:**
  * *Алгоритмы:* `sort` (TimSort **O(N log N)**), `binarySearch`, `frequency`, `disjoint`. Метод `addAll` работает быстрее стандартного.
  * *Обертки:* `unmodifiableList` (защита от записи, но отражает изменения оригинала), `synchronizedList` (потокобезопасность через мьютекс).
  * *Checked Collections:* `checkedList()` — динамическая проверка типов для защиты кучи от legacy-кода с сырыми типами.
* **Класс `java.util.Arrays`:**
  * Утилиты для примитивов и объектов. `asList()` создает View фиксированного размера.
  * *Многомерные массивы:* `deepEquals()` и `deepToString()` для рекурсивной обработки матриц.
  * *Мост к Stream API:* `Arrays.stream()`.
* **Неизменяемые (Immutable) фабрики (Java 9+):**
  * `List.of()`, `Set.of()`, `Map.of()`. Абсолютно неизменяемы, запрещают `null`. Идеальны для DTO и констант.
  * `List.copyOf()` (Java 10+) для безопасного снятия слепка с изменяемых коллекций.
